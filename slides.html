Modern HTML Reporting in R

No Shiny needed

Dylan Lewis

Motivation & Context

Reporting Constraints in PHS

  • Governance and hosting constraints often block Shiny.
  • However, teams still need interactive, attractive outputs.
  • Requirements:
    • Lightweight HTML files
    • Reproducible, maintainable workflows
    • Accessible and long‑lived outputs

Dashboards vs Parameterised Reports

  • Quickly communicate a variety of data at a glance
  • Monitor live or fast-changing statistics

  • In-depth exploration of data
  • Can be be ‘parameterised’

Authoring Formats for HTML Reporting

Quarto & R Markdown

Both generate high‑quality HTML reports, but with different strengths.

Why Quarto?

  • Modern defaults
  • Long-term support
  • Cross-language capabilities
  • Built‑in brand/themes/includes

Why R Markdown?

  • Mature ecosystem
  • Some tooling better documented
  • Familiar across R users

Parameterised Reports (Quarto & RMd)

  • Both systems support parameters, which should be defined in the YAML header.
  • Use parameters to batch reports across sites/boards, rerun for different time periods, etc.
  • Access with params$....

Leaving Markdown Behind

You can also build HTML directly using bslib + htmltools:

page_fluid(
  H1("Penguins"),
  card(
    datatable(penguins)
  )
)

Benefits:

  • Full layout flexibility
  • Multi‑page designs
  • Reusable components

Trade‑off:

  • Less author-friendly than markdown

Example: Multi-page Custom UI

# pages
flipper_length <- nav_panel(
  "Flipper length",
  layout_columns(
    col_widths = breakpoints(
      xs = c(-2,8,-2), xxl = c(-3,6,-3)),
    "This page examines the flipper length of penguins",
    navset_pill(
      nav_panel("Male", 
                card_body(
                  fillable=FALSE,
                  my_plot("male"))),
      nav_panel("Female",
                card_body(
                  fillable=FALSE,
                  my_plot("female")))
    ))
  )

# report
page_navbar(
  title = "Penguin report",
  fillable = FALSE,
  navbar_options = navbar_options(
    bg = "#3F3685"
  ),
  flipper_length
)

Saving Standalone HTML from R

  • Exporting self‑contained HTML outputs can be fiddly…
  • Images need to be encoded in base64
img_base64 <- function(path, ...) {
  base64 <- base64enc::dataURI(file = path, mime = "image/png")
  htmltools::tags$img(src = base64, ...)
}
  • Fonts should be bundled to ensure consistency
gdtools::addGFontHtmlDependency(family = "Open Sans")
  • htmlwidgets::saveWidget() works in some cases, but I have a custom save_self_contained_html() function that I’m happy to share

Adding Interactivity Without Shiny

UI Components

bslib package can mimic Shiny-like behavoiur in a static document.

For example, hidden nav containers can be used to select between filtered plots (in lieu of responsive inputs):

my_plot <- function(sx){
  girafe(
    filter(penguins, sex == sx) |>
      ggplot(aes(body_mass, flipper_len)) +
      geom_point()
  )
}

navset_pill(
  nav_panel("Male", 
            card_body(
              fillable=FALSE,
              my_plot("male"))
  ),
  nav_panel("Female",
            card_body(
              fillable=FALSE,
              my_plot("female"))
  )
)

Interactive Visuals: ggiraph

  • Why ggiraph?
    • Lightweight SVG output.
    • Plotly for R no longer officially supported.
    • Easier to customise and control (no messing around with JS)

Example

plot <- ggplot(
  penguins,
  aes( x= body_mass, y = flipper_len, colour = species,
       tooltip = paste0(
         "Year: ", year),
       data_id = year
       )) +
  geom_point_interactive() +
  theme_light()

girafe(plot,
       options = list(opts_hover(css = "opacity:0.9"),
                      opts_hover_inv(css = "opacity:0.3")))
1
Tooltip can be a variable, string, or glue function
2
data_id controls hover effect groups
3
Most ggplot2 geoms have an _interactive variant
4
Unlike plotly, ggplot2 themes work without issues
5
Call girafe() to turn any ggplot object into an SVG widget
6
ggiraph allows the user to define the interactive behaviour

Interactive Tables: DT + Extensions

  • DT extensions give more options for search, filtering, and export buttons.
  • E.g. SearchPanes extension:
penguins |>
  datatable(extensions = c('Select', 'SearchPanes'),
              selection = 'none',
              rownames = FALSE,
              options = list(dom = 'Prtip',
                             searchPanes = list(
                               controls = FALSE,
                               viewCount = FALSE
                             ),
                             columnDefs = list(
                               list(searchPanes = list(show = FALSE),
                                    targets = 2:6)
                             ))
            )
